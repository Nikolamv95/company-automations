# CRO Checklist & Conversion Rules

Apply to every landing page before delivery.

---

## Above-Fold Rules (Most Critical)

The above-fold area (visible before scrolling) must communicate the full value proposition.

```
✓ Headline visible without scrolling on mobile (375px)
✓ Subheadline visible without scrolling
✓ CTA button visible without scrolling
✓ Value prop understood in under 5 seconds
✓ No navigation menu (removes exit paths)
✓ No unrelated links above fold
```

**Test:** Show it to someone for 5 seconds. Ask: "What is this page trying to do?" They should answer correctly.

---

## Friction Reduction

Every extra element is friction. Every friction point costs conversions.

**Form fields:**
- Each field reduces completion by 7-10%
- Opt-in: first name + email only (2 fields max)
- Sales page: name + email + card (3 fields max)
- Never ask for phone number unless legally required
- Never ask for address before payment confirmed

**CTAs:**
- One CTA per page section (multiple sections can have CTAs, but each section has only one)
- CTA buttons: benefit-led, not action-led
  - ✅ "Get My Free Guide" / "Start Losing the Bloating" / "Claim My Discount"
  - ❌ "Submit" / "Buy Now" / "Click Here"

**Navigation:**
- Sales pages: remove all navigation menu items
- Opt-in pages: remove all navigation menu items
- Checkout: no navigation, no exit links

## Social Proof Placement

Proof must appear at two critical moments:
1. **Near top** — before the visitor scrolls (to earn trust early)
2. **Near CTA** — immediately before the purchase decision

**Proof hierarchy (most to least powerful):**
1. Video testimonials with specific results
2. Before/after photos with named customer and result
3. Text testimonials with specific numbers, name, and timeframe
4. Star ratings with review count
5. Media logos ("As seen in...")
6. Customer count ("Join 4,200 customers")

**Specificity rule:** Always use numbers.
- ❌ "She lost a lot of weight"
- ✅ "She lost 23 lbs in 6 weeks without changing her diet"

---

## Price Framing

Never reveal price without anchoring first.

**Anchoring sequence:**
1. Show value stack (what they're getting + individual values)
2. Add up total value: "If purchased separately: $[X]"
3. Reveal price: "Today only: $[Y]"
4. Frame daily rate: "That's just $[Z]/day — less than a coffee"
5. Add guarantee: "And if it doesn't work: full refund, no questions"

**Price display:**
- Original price crossed out (if applicable)
- Current price bold and prominent
- Guarantee badge or text immediately below price

---

## Objection Handling — Placement Rules

Handle the top 3 objections BEFORE the buy button. Common objections for supplement brands:

| Objection | Where to handle |
|-----------|----------------|
| "How is this different?" | Mechanism section (Section 4) |
| "What if it doesn't work?" | Guarantee section (near price) |
| "How long until I see results?" | Proof section + FAQ |
| "Is this safe?" | FAQ |
| "Can I get a refund?" | Guarantee section + checkout page |

## Mobile Optimization

Most supplement buyers on Facebook/TikTok traffic are on mobile.

```
✓ CTA button: thumb-reachable (bottom 30% of screen on mobile)
✓ Button size: minimum 44px height
✓ Text size: minimum 16px body, 24px+ headlines
✓ No horizontal scroll
✓ Images load under 3 seconds
✓ Form fields: large enough to tap without zooming
```

---

## Page Speed

Every second of load time reduces conversions by ~7%.

```
✓ Page loads under 3 seconds on mobile 3G
✓ Images compressed (WebP format preferred)
✓ No autoplay video above fold (blocks load)
✓ Minimal third-party scripts above fold
```

## Headline Formulas

Match to awareness level:

```
Level 1-2 (Unaware / Problem Aware):
  "Why [frustrating thing] keeps happening — and what to do about it"
  "The real reason [common problem] won't go away (it's not what you think)"

Level 3 (Solution Aware):
  "The [mechanism] that [result] — without [sacrifice]"
  "How [avatar] finally [result] after [failed attempts]"

Level 4 (Product Aware):
  "Join [X] people who [result] in [timeframe] with [product]"
  "[Specific result] — or your money back"

Level 5 (Most Aware):
  "[Offer]: [Price]. [Guarantee]. [CTA]."
  Direct and immediate — they already know they want it
```

## Pre-Launch Page Audit Checklist

Run before every page goes live:

**Above fold:**
- [ ] Headline communicates primary desire
- [ ] CTA visible without scrolling (mobile)
- [ ] Value prop understood in 5 seconds

**Social proof:**
- [ ] At least one proof element above fold
- [ ] At least one proof element near CTA

**Copy:**
- [ ] No sentences over 20 words
- [ ] No jargon or marketing speak
- [ ] Specific numbers throughout (not "many" or "some")
- [ ] 5th grade reading level

**Offer:**
- [ ] Guarantee stated specifically (days + terms)
- [ ] Price anchored with value stack
- [ ] Top 3 objections handled before CTA

**Technical:**
- [ ] No navigation menu (sales/checkout)
- [ ] No exit links near CTA
- [ ] Mobile thumb-reachable CTA button
- [ ] Page loads under 3 seconds
