# Landing Page Templates

Full section-by-section templates for each page type. Read before writing any landing page.

---

## Template 1 — Opt-In Page (100-200 words above fold)

Goal: capture email in exchange for a lead magnet. One action only.

```
HEADLINE        ← primary desire in customer's language (6-12 words)
                   Formula: "[Specific outcome] without [sacrifice]"
                   Example: "Lose the bloating without giving up your favorite foods"

SUBHEADLINE     ← expand the promise + hint at mechanism (1-2 sentences)
                   "Discover the [mechanism] that [result] — even if you've [failed before]"

BULLET LIST     ← 3-5 specific outcomes (benefit-forward, not feature-forward)
                   ✓ [Specific result they get]
                   ✓ [Pain removed]
                   ✓ [Surprise benefit]

FORM            ← First name + Email only
                   Label the button with outcome, not action:
                   ✅ "Send Me the Guide"
                   ❌ "Submit"

SOCIAL PROOF    ← below the form: subscriber count, quote, or logo bar
```

**Above-fold rule:** Everything above must be visible without scrolling on mobile. Test on 375px width.

## Template 2 — Long-Form Sales Page (1500-3000 words)

Goal: convert cold or warm traffic to purchase. Full RMBC structure.

### Section 1: Headline + Subheadline
```
HEADLINE     ← awareness-appropriate, primary desire
             Level 1-2: Problem-led ("Why [frustrating thing] keeps happening after [event]")
             Level 3:   Mechanism-led ("The [mechanism] that finally [result]")
             Level 4-5: Direct/proof ("Join [X] people who [result] in [timeframe]")

SUBHEADLINE  ← expand promise, hint at differentiation
             "Without [sacrifice]. Even if you've tried [common solution] before."
```

### Section 2: Hook Story (mirror their pain)
```
Open with the avatar's worst moment — specific, sensory, real.
"She stood in the dressing room..."
"He'd been taking the same supplement for 6 months..."

Pull them into recognition: "Sound familiar?"

Bridge to: "There's a reason this keeps happening. And it's not what you think."
```

### Section 3: Failed Attempts (validate the struggle)
```
Name every category they've tried:
"You've probably tried [X]."
"You've heard about [Y]."
"Maybe you've even done [Z]."

Each with a 1-line reason why it doesn't work:
"The problem with [X] is that it doesn't address [root cause]."

End with: "It's not your fault. The real cause is [mechanism tease]."
```

### Section 4: Mechanism (the discovery)
```
Name the mechanism. Give it a clear, memorable label.
"It all comes down to [mechanism name]."

Explain in 5th grade language:
- What it is (1-2 sentences)
- Why it causes the problem (1-2 sentences)
- Why common solutions miss it (1 sentence)
```

### Section 5: Proof Section
```
3-5 testimonials that mirror the avatar's journey:
"[Name] struggled with [specific problem] for [time]. [What they tried]. [What changed]. [Specific result with numbers]."

Add: clinical backing if applicable (ingredient studies, not product claims)
Add: before/after imagery if available
```

### Section 6: What You Get
```
Lead with benefits, not features:
❌ "60 capsules per bottle"
✅ "A full month's supply — enough to see the results for yourself"

For each element of the offer:
→ [What it is] — [What it does for them] — [Why it matters]
```

### Section 7: For Who / Not For Who
```
"This is for you if:"
- [Avatar qualifier 1]
- [Avatar qualifier 2]
- [Avatar qualifier 3]

"This is NOT for you if:"
- [Disqualifier — sets expectations, builds trust]
```

### Section 8: Objection Handling
```
Top 3 objections, direct Q&A format:
Q: "How is this different from [what I've already tried]?"
A: [Direct answer that references the mechanism]

Q: "What if it doesn't work for me?"
A: [Guarantee response]

Q: "How long before I see results?"
A: [Specific timeframe with honest caveat]
```

### Section 9: Pricing + Value Stack
```
1. Show what they're getting (list all elements)
2. Assign value to each: "If you bought each piece separately..."
3. Anchor total: "Total value: $[X]"
4. Reveal price: "Today: $[Y]"
5. Frame daily: "That's less than $[Z]/day"
```

### Section 10: Guarantee
```
Make it specific and generous:
✅ "60-day money-back guarantee. If you're not satisfied for any reason — even if you just changed your mind — email us and get every penny back. No questions, no hassle."

❌ "Satisfaction guaranteed"
```

### Section 11: Final CTA + Urgency
```
Restate the transformation in one sentence.
State the urgency reason (real, not fake):
"[Bonus] expires [date]" / "Current price locks in only for [timeframe]"

[Primary CTA button — benefit-led]
[Guarantee reminder line below button]
```

### Section 12: FAQ
```
5-7 questions that handle remaining micro-objections:
- Shipping and delivery
- Dosage and usage
- Who it's for / not for (reinforce)
- Returns (reinforce guarantee)
- Expected timeline (reinforce proof)
```

## Template 3 — Checkout / Order Form Page

Goal: minimize abandonment at the final step.

```
HEADLINE          ← reinforce the decision ("You're almost in!")
                     Not: "Order Page" / Not: product name only

ORDER SUMMARY     ← clear, simple: what they're getting + price
                     No upsell confusion before payment fields

TRUST SIGNALS     ← lock icon (SSL), payment logos, guarantee badge
                     All visible before form fields

OBJECTION HANDLER ← 1-2 sentences before payment fields
                     "Remember: you're protected by our [X]-day guarantee."

FORM FIELDS       ← name, email, card only — nothing extra

CTA BUTTON        ← "Complete My Order" / "Yes, I Want This" / "Claim My [Product]"
                     NOT "Submit" or "Buy Now"

GUARANTEE REPEAT  ← visible next to or below submit button
```

**Checkout rules:**
- No exit links — every link that isn't the buy button is a conversion leak
- No navigation menu
- Trust signals must be above the fold on mobile
- One upsell maximum — and only after payment is complete (not on this page)
